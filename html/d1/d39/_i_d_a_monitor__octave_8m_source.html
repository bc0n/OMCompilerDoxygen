1 function [new_data] = IDAMonitor(call, T, Y, YQ, YS, data)
2 %IDAMonitor is the default IDAS monitoring function.
3 % To use it, set the Monitor property in IDASetOptions to
4 % 'IDAMonitor' or to @IDAMonitor and 'MonitorData' to mondata
5 % (defined as a structure).
6 %
7 % With default settings, this function plots the evolution of the step
8 % size, method order, and various counters.
9 %
10 % Various properties can be changed from their default values by passing
11 % to IDASetOptions, through the property 'MonitorData', a structure
12 % MONDATA with any of the following fields. If a field is not defined,
13 % the corresponding default value is used.
14 %
15 % Fields in MONDATA structure:
16 % o stats [ {true} | false ]
17 % If true, report the evolution of the step size and method order.
18 % o cntr [ {true} | false ]
19 % If true, report the evolution of the following counters:
20 % nst, nfe, nni, netf, ncfn (see IDAGetStats)
21 % o sol [ true | {false} ]
22 % If true, plot solution components.
23 % o sensi [ true | {false} ]
24 % If true and if FSA is enabled, plot sensitivity components.
25 % o select [ array of integers ]
26 % To plot only particular solution components, specify their indeces in
27 % the field select. If not defined, but sol=true, all components are plotted.
28 % o updt [ integer | {50} ]
29 % Update frequency. Data is posted in blocks of dimension n.
30 % o skip [ integer | {0} ]
31 % Number of integrations steps to skip in collecting data to post.
32 % o post [ {true} | false ]
33 % If false, disable all posting. This option is necessary to disable
34 % monitoring on some processors when running in parallel.
35 %
36 % See also IDASetOptions, IDAMonitorFn
37 %
38 % NOTES:
39 % 1. The argument mondata is REQUIRED. Even if only the default options
40 % are desired, set mondata=struct; and pass it to IDASetOptions.
