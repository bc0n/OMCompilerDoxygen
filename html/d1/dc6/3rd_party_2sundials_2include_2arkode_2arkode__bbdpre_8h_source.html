18  * This is the header file for the ARKBBDPRE module, for a
19  * band-block-diagonal preconditioner, i.e. a block-diagonal
20  * matrix with banded blocks, for use with ARKSPGMR, ARKSPBCG,
21  * ARKSPTFQMR, ARKSPFGMR or ARKPCG, and the parallel implementation
22  * of the NVECTOR module.
23  *
24  * Summary:
25  *
26  * These routines provide a preconditioner matrix that is
27  * block-diagonal with banded blocks. The blocking corresponds
28  * to the distribution of the dependent variable vector y among
29  * the processors. Each preconditioner block is generated from
30  * the Jacobian of the local part (on the current processor) of a
31  * given function g(t,y) approximating f(t,y). The blocks are
32  * generated by a difference quotient scheme on each processor
33  * independently. This scheme utilizes an assumed banded
34  * structure with given half-bandwidths, mudq and mldq.
35  * However, the banded Jacobian block kept by the scheme has
36  * half-bandwiths mukeep and mlkeep, which may be smaller.
37  *
38  * The user's calling program should have the following form:
39  *
40  * #include <nvector_parallel.h>
41  * #include <arkode/arkode_bbdpre.h>
42  * ...
43  * void *arkode_mem;
44  * ...
45  * Set y0
46  * ...
47  * arkode_mem = ARKodeCreate();
48  * ier = ARKodeInit(...);
49  * ...
50  * flag = ARKSpgmr(arkode_mem, pretype, maxl);
51  * -or-
52  * flag = ARKSpbcg(arkode_mem, pretype, maxl);
53  * -or-
54  * flag = ARKSptfqmr(arkode_mem, pretype, maxl);
55  * -or-
56  * flag = ARKSpfgmr(arkode_mem, pretype, maxl);
57  * -or-
58  * flag = ARKPcg(arkode_mem, pretype, maxl);
59  * ...
60  * flag = ARKBBDPrecInit(arkode_mem, Nlocal, mudq ,mldq,
61  * mukeep, mlkeep, dqrely, gloc, cfn);
62  * ...
63  * ier = ARKode(...);
64  * ...
65  * ARKodeFree(&arkode_mem);
66  *
67  * Free y0
68  *
69  * The user-supplied routines required are:
70  *
71  * f = function defining the ODE right-hand side f(t,y).
72  *
73  * gloc = function defining the approximation g(t,y).
74  *
75  * cfn = function to perform communication need for gloc.
76  *
77  * Notes:
78  *
79  * 1) This header file is included by the user for the definition
80  * of the ARKBBDData type and for needed function prototypes.
81  *
82  * 2) The ARKBBDPrecInit call includes half-bandwiths mudq and mldq
83  * to be used in the difference quotient calculation of the
84  * approximate Jacobian. They need not be the true
85  * half-bandwidths of the Jacobian of the local block of g,
86  * when smaller values may provide a greater efficiency.
87  * Also, the half-bandwidths mukeep and mlkeep of the retained
88  * banded approximate Jacobian block may be even smaller,
89  * to reduce storage and computation costs further.
90  * For all four half-bandwidths, the values need not be the
91  * same on every processor.
92  *
93  * 3) The actual name of the user's f function is passed to
94  * ARKodeInit, and the names of the user's gloc and cfn
95  * functions are passed to ARKBBDPrecInit.
96  *
97  * 4) The pointer to the user-defined data block user_data, which is
98  * set through ARKodeSetUserData is also available to the user in
99  * gloc and cfn.
100  *
101  * 5) Optional outputs specific to this module are available by
102  * way of routines listed below. These include work space sizes
103  * and the cumulative number of gloc calls. The costs
104  * associated with this module also include nsetups banded LU
105  * factorizations, nlinsetups cfn calls, and npsolves banded
106  * backsolve calls, where nlinsetups and npsolves are
107  * integrator/ARKSPGMR/ARKSPBCG/ARKSPTFQMR/ARKSPFGMR/ARKPCG
108  * optional outputs.
109  *--------------------------------------------------------------*/

120 /*---------------------------------------------------------------
121  Type: ARKLocalFn
122 
123  The user must supply a function g(t,y) which approximates the
124  right-hand side function f for the system y'=f(t,y), and which
125  is computed locally (without interprocess communication).
126  (The case where g is mathematically identical to f is allowed.)
127  The implementation of this function must have type ARKLocalFn.
128 
129  This function takes as input the local vector size Nlocal, the
130  independent variable value t, the local real dependent
131  variable vector y, and a pointer to the user-defined data
132  block user_data. It is to compute the local part of g(t,y) and
133  store this in the vector g.
134  (Allocation of memory for y and g is handled within the
135  preconditioner module.)
136  The user_data parameter is the same as that specified by the user
137  through the ARKodeSetFdata routine.
138 
139  A ARKLocalFn should return 0 if successful, a positive value if
140  a recoverable error occurred, and a negative value if an
141  unrecoverable error occurred.
142 ---------------------------------------------------------------*/
143 typedef int (*ARKLocalFn)(long int Nlocal, realtype t, N_Vector y,
144  N_Vector g, void *user_data);

147 /*---------------------------------------------------------------
148  Type: ARKCommFn
149 
150  The user may supply a function of type ARKCommFn which performs
151  all interprocess communication necessary to evaluate the
152  approximate right-hand side function described above.
153 
154  This function takes as input the local vector size Nlocal,
155  the independent variable value t, the dependent variable
156  vector y, and a pointer to the user-defined data block user_data.
157  The user_data parameter is the same as that specified by the user
158  through the ARKodeSetUserData routine. The ARKCommFn cfn is
159  expected to save communicated data in space defined within the
160  structure user_data. Note: A ARKCommFn cfn does not have a return value.
161 
162  Each call to the ARKCommFn cfn is preceded by a call to the
163  ARKRhsFn f with the same (t,y) arguments. Thus cfn can omit any
164  communications done by f if relevant to the evaluation of g.
165  If all necessary communication was done by f, the user can
166  pass NULL for cfn in ARKBBDPrecInit (see below).
167 
168  A ARKCommFn should return 0 if successful, a positive value if
169  a recoverable error occurred, and a negative value if an
170  unrecoverable error occurred.
171 ---------------------------------------------------------------*/
172 typedef int (*ARKCommFn)(long int Nlocal, realtype t,
173  N_Vector y, void *user_data);

176 /*---------------------------------------------------------------
177  ARKBBDPrecInit:
178 
179  ARKBBDPrecInit allocates and initializes the BBD preconditioner.
180 
181  The parameters of ARKBBDPrecInit are as follows:
182 
183  arkode_mem is the pointer to the integrator memory.
184 
185  Nlocal is the length of the local block of the vectors y etc.
186  on the current processor.
187 
188  mudq, mldq are the upper and lower half-bandwidths to be used
189  in the difference quotient computation of the local
190  Jacobian block.
191 
192  mukeep, mlkeep are the upper and lower half-bandwidths of the
193  retained banded approximation to the local Jacobian
194  block.
195 
196  dqrely is an optional input. It is the relative increment
197  in components of y used in the difference quotient
198  approximations. To specify the default, pass 0.
199  The default is dqrely = sqrt(unit roundoff).
200 
201  gloc is the name of the user-supplied function g(t,y) that
202  approximates f and whose local Jacobian blocks are
203  to form the preconditioner.
204 
205  cfn is the name of the user-defined function that performs
206  necessary interprocess communication for the
207  execution of gloc.
208 
209  The return value of ARKBBDPrecInit is one of:
210  ARKSPILS_SUCCESS if no errors occurred
211  ARKSPILS_MEM_NULL if the integrator memory is NULL
212  ARKSPILS_LMEM_NULL if the linear solver memory is NULL
213  ARKSPILS_ILL_INPUT if an input has an illegal value
214  ARKSPILS_MEM_FAIL if a memory allocation request failed
215 ---------------------------------------------------------------*/
216 SUNDIALS_EXPORT int ARKBBDPrecInit(void *arkode_mem, long int Nlocal,
217  long int mudq, long int mldq,
218  long int mukeep, long int mlkeep,
