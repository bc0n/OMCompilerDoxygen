104 Purpose:
105 
106  Given the nonzero pattern of a sparse matrix A in column-oriented form, and
107  a sparsity preserving column pre-ordering Qinit, umfpack_*_qsymbolic
108  performs the symbolic factorization of A*Qinit (or A (:,Qinit) in MATLAB
109  notation). This is identical to umfpack_*_symbolic, except that neither
110  COLAMD nor AMD are called and the user input column order Qinit is used
111  instead. Note that in general, the Qinit passed to umfpack_*_qsymbolic
112  can differ from the final Q found in umfpack_*_numeric. The unsymmetric
113  strategy will perform a column etree postordering done in
114  umfpack_*_qsymbolic and sparsity-preserving modifications are made within
115  each frontal matrix during umfpack_*_numeric. The symmetric and 2-by-2
116  strategies will preserve Qinit, unless the matrix is structurally singular.
117 
118  See umfpack_*_symbolic for more information.
119 
120  *** WARNING *** A poor choice of Qinit can easily cause umfpack_*_numeric
121  to use a huge amount of memory and do a lot of work. The "default" symbolic
122  analysis method is umfpack_*_symbolic, not this routine. If you use this
123  routine, the performance of UMFPACK is your responsibility; UMFPACK will
124  not try to second-guess a poor choice of Qinit.
125 
126 Returns:
127 
128  The value of Info [UMFPACK_STATUS]; see umfpack_*_symbolic.
129  Also returns UMFPACK_ERROR_invalid_permuation if Qinit is not a valid
130  permutation vector.
131 
132 Arguments:
133 
134  All arguments are the same as umfpack_*_symbolic, except for the following:
135 
136  Int Qinit [n_col] ; Input argument, not modified.
137 
138  The user's fill-reducing initial column pre-ordering. This must be a
139  permutation of 0..n_col-1. If Qinit [k] = j, then column j is the kth
140  column of the matrix A (:,Qinit) to be factorized. If Qinit is an
141  (Int *) NULL pointer, then COLAMD or AMD are called instead.
142 
143  double Control [UMFPACK_CONTROL] ; Input argument, not modified.
144 
145  If Qinit is not NULL, then only two strategies are recognized:
146  the unsymmetric strategy and the symmetric strategy.
147  If Control [UMFPACK_STRATEGY] is UMFPACK_STRATEGY_SYMMETRIC,
148  then the symmetric strategy is used. Otherwise the unsymmetric
149  strategy is used.
150 */
