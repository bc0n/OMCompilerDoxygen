23 /*
24  * ==============================================================================
25  *
26  * FIDABBD Interface Package
27  *
28  * The FIDABBD Interface Package is a package of C functions which,
29  * together with the FIDA Interface Package, support the use of the
30  * IDA solver (parallel MPI version) with the IDABBDPRE preconditioner module,
31  * for the solution of DAE systems in a mixed Fortran/C setting. The
32  * combination of IDA and IDABBDPRE solves DAE systems with the SPGMR
33  * (scaled preconditioned GMRES), SPBCG (scaled preconditioned Bi-CGSTAB), or
34  * SPTFQMR (scaled preconditioned TFQMR) method for the linear systems that arise,
35  * and with a preconditioner that is block-diagonal with banded blocks. While
36  * IDA and IDABBDPRE are written in C, it is assumed here that the user's
37  * calling program and user-supplied problem-defining routines are written in
38  * Fortran.
39  *
40  * The user-callable functions in this package, with the corresponding
41  * IDA and IDABBDPRE functions, are as follows:
42  * FIDABBDININT interfaces to IDABBDPrecInit
43  * FIDABBDREINIT interfaces to IDABBDPrecReInit
44  * FIDABBDOPT accesses optional outputs
45  * FIDABBDFREE interfaces to IDABBDPrecFree
46  *
47  * In addition to the Fortran residual function FIDARESFUN, the
48  * user-supplied functions used by this package, are listed below,
49  * each with the corresponding interface function which calls it (and its
50  * type within IDABBDPRE or IDA):
51  * FIDAGLOCFN is called by the interface function FIDAgloc of type IDABBDLocalFn
52  * FIDACOMMFN is called by the interface function FIDAcfn of type IDABBDCommFn
53  * FIDAJTIMES (optional) is called by the interface function FIDAJtimes of
54  * type IDASpilsJacTimesVecFn
55  * (The names of all user-supplied routines here are fixed, in order to
56  * maximize portability for the resulting mixed-language program.)
57  *
58  * Important note on portability:
59  * In this package, the names of the interface functions, and the names of
60  * the Fortran user routines called by them, appear as dummy names
61  * which are mapped to actual values by a series of definitions in the
62  * header file fidabbd.h.
63  *
64  * ==============================================================================
65  *
66  * Usage of the FIDA/FIDABBD Interface Packages
67  *
68  * The usage of the combined interface packages FIDA and FIDABBD requires
69  * calls to several interface functions, and a few different user-supplied
70  * routines which define the problem to be solved and indirectly define
71  * the preconditioner. These function calls and user routines are
72  * summarized separately below.
73  *
74  * Some details are omitted, and the user is referred to the IDA user document
75  * for more complete information.
76  *
77  * (1) User-supplied residual routine: FIDARESFUN
78  * The user must in all cases supply the following Fortran routine
79  * SUBROUTINE FIDARESFUN(T, Y, YP, R, IPAR, RPAR, IER)
80  * DIMENSION Y(*), YP(*), R(*), IPAR(*), RPAR(*)
81  * It must set the R array to F(t,y,y'), the residual of the DAE
82  * system, as a function of T = t, the array Y = y, and the array YP = y'.
83  * Here Y, YP and R are distributed vectors.
84  *
85  * (2) User-supplied routines to define preconditoner: FIDAGLOCFN and FIDACOMMFN
86  *
87  * The routines in the IDABBDPRE module provide a preconditioner matrix
88  * for IDA that is block-diagonal with banded blocks. The blocking
89  * corresponds to the distribution of the dependent variable vectors y and y'
90  * among the processes. Each preconditioner block is generated from the
91  * Jacobian of the local part (associated with the current process) of a given
92  * function G(t,y,y') approximating F(t,y,y'). The blocks are generated by a
93  * difference quotient scheme independently by each process, utilizing
94  * an assumed banded structure with given half-bandwidths. A separate
95  * pair of half-bandwidths defines the band matrix retained.
96  *
97  * (2.1) Local approximate function FIDAGLOCFN.
98  * The user must supply a subroutine of the form
99  * SUBROUTINE FIDAGLOCFN(NLOC, T, YLOC, YPLOC, GLOC, IPAR, RPAR, IER)
100  * DIMENSION YLOC(*), YPLOC(*), GLOC(*), IPAR(*), RPAR(*)
101  * to compute the function G(t,y,y') which approximates the residual
102  * function F(t,y,y'). This function is to be computed locally, i.e., without
103  * interprocess communication. (The case where G is mathematically
104  * identical to F is allowed.) It takes as input the local vector length
105  * NLOC, the independent variable value T = t, and the local realtype
106  * dependent variable arrays YLOC and YPLOC. It is to compute the local part
107  * of G(t,y,y') and store this in the realtype array GLOC.
108  *
109  * (2.2) Communication function FIDACOMMF.
110  * The user must also supply a subroutine of the form
111  * SUBROUTINE FIDACOMMFN(NLOC, T, YLOC, YPLOC, IPAR, RPAR, IER)
112  * DIMENSION YLOC(*), YPLOC(*), IPAR(*), RPAR(*)
113  * which is to perform all interprocess communication necessary to
114  * evaluate the approximate residual function G described above.
115  * This function takes as input the local vector length NLOC, the
116  * independent variable value T = t, and the local real dependent
117  * variable arrays YLOC and YPLOC. It is expected to save communicated
118  * data in work space defined by the user, and made available to FIDAGLOCFN.
119  * Each call to the FIDACOMMFN is preceded by a call to FIDARESFUN with
120  * the same (t,y,y') arguments. Thus FIDACOMMFN can omit any
121  * communications done by FIDARESFUN if relevant to the evaluation of G.
122  *
123  * (3) Optional user-supplied Jacobian-vector product routine: FIDAJTIMES
124  * As an option when using the SPGMR/SPBCG/SPTFQMR linear solver, the user may
125  * supply a routine that computes the product of the system Jacobian J = df/dy
126  * and a given vector v. If supplied, it must have the following form:
127  * SUBROUTINE FIDAJTIMES(T, Y, YP, R, V, FJV, CJ, EWT, H,
128  * 1 IPAR, RPAR, WK1, WK2, IER)
129  * DIMENSION V(*), FJV(*), Y(*), YP(*), R(*), EWT(*),
130  * 1 , IPAR(*), RPAR(*), WK1(*), WK2(*)
131  * This routine must compute the product vector Jv, where the vector v is stored
132  * in V, and store the product in FJV. On return, set IER = 0 if FIDAJTIMES was
133  * successful, and nonzero otherwise.
134  *
135  * (4) Initialization: FNVINITP, FIDAMALLOC, FIDABBDINIT.
136  *
137  * (4.1) To initialize the parallel machine environment, the user must make
138  * one of the following calls:
139  * CALL FNVINITP (KEY, NLOCAL, NGLOBAL, IER)
140  * -or-
141  * CALL FNVINITP (COMM, KEY, NLOCAL, NGLOBAL, IER)
142  * The arguments are:
143  * COMM = MPI communicator (e.g., MPI_COMM_WORLD)
144  * KEY = 3 for IDA
145  * NLOCAL = local size of vectors on this processor
146  * NGLOBAL = the system size, and the global size of vectors (the sum
147  * of all values of NLOCAL)
148  * IER = return completion flag. Values are 0 = success, -1 = failure.
149  * NOTE: The COMM argument passed to the FNVINITP routine is only supported if
150  * the MPI implementation used to build SUNDIALS includes the MPI_Comm_f2c
151  * function from the MPI-2 specification. To check if the function is supported
152  * look for the line "#define SUNDIALS_MPI_COMM_F2C 1" in the sundials_config.h
153  * header file.
154  *
155  * (4.2) To set various problem and solution parameters and allocate
156  * internal memory, make the following call:
157  * CALL FIDAMALLOC(T0, Y0, YP0, IATOL, RTOL, ATOL, ID, CONSTR,
158  * 1 IOUT, ROUT, IPAR, RPAR, IER)
159  * The arguments are:
160  * T0 = initial value of t
161  * Y0 = array of initial conditions, y(t0)
162  * YP0 = value of y'(t0)
163  * IATOL = type for absolute tolerance ATOL: 1 = scalar, 2 = array.
164  * If IATOL = 3, then the user must supply a routine FIDAEWT to compute
165  * the error weight vector.
166  * RTOL = relative tolerance (scalar)
167  * ATOL = absolute tolerance (scalar or array)
168  * IOUT = array of length at least 21 for integer optional inputs and outputs
169  * (declare as INTEGER*4 or INTEGER*8 according to C type long int)
170  * ROUT = array of length 6 for real optional inputs and outputs
171  *
172  * The optional outputs are:
173  *
174  * LENRW = IOUT( 1) -> IDAGetWorkSpace
175  * LENIW = IOUT( 2) -> IDAGetWorkSpace
176  * NST = IOUT( 3) -> IDAGetNumSteps
177  * NRE = IOUT( 4) -> IDAGetNumResEvals
178  * NETF = IOUT( 5) -> IDAGetNumErrTestFails
179  * NCFN = IOUT( 6) -> IDAGetNumNonlinSolvConvFails
180  * NNI = IOUT( 7) -> IDAGetNumNonlinSolvIters
181  * NSETUPS = IOUT( 8) -> IDAGetNumLinSolvSetups
182  * KLAST = IOUT( 9) -> IDAGetLastOrder
183  * KCUR = IOUT(10) -> IDAGetCurrentOrder
184  * NBCKTRK = IOUT(11) -> IDAGetNumBacktrackOps
185  * NGE = IOUT(12) -> IDAGetNumGEvals
186  *
187  * HINUSED = ROUT( 1) -> IDAGetActualInitStep
188  * HLAST = ROUT( 2) -> IDAGetLastStep
189  * HCUR = ROUT( 3) -> IDAGetCurrentStep
190  * TCUR = ROUT( 4) -> IDAGetCurrentTime
191  * TOLSFAC = ROUT( 5) -> IDAGetTolScaleFactor
192  * UNITRND = ROUT( 6) -> UNIT_ROUNDOFF
193  *
194  * IPAR = array with user integer data
195  * (declare as INTEGER*4 or INTEGER*8 according to C type long int)
196  * RPAR = array with user real data
197  * IER = return completion flag. Values are 0 = SUCCESS, and -1 = failure.
198  * See printed message for details in case of failure.
199  *
200  * If the user program includes the FIDAEWT routine for the evaluation of the
201  * error weights, the following call must be made
202  * CALL FIDAEWTSET (FLAG, IER)
203  * with FLAG = 1 to specify that FIDAEWT is provided.
204  * The return flag IER is 0 if successful, and nonzero otherwise.
205  *
206  * (4.3) Attach one of the 3 SPILS linear solvers. Make one of the
207  * following calls (see fida.h for more details).
208  * CALL FIDASPGMR(MAXL, IGSTYPE, MAXRS, EPLIFAC, DQINCFAC, IER)
209  * CALL FIDASPBCG(MAXL, EPLIFAC, DQINCFAC, IER)
210  * CALL FIDASPTFQMR(MAXL, EPLIFAC, DQINCFAC, IER)
211  *
212  * (4.4) To allocate memory and initialize data associated with the IDABBDPRE
213  * preconditioner, make the following call:
214  * CALL FIDABBDINIT(NLOCAL, MUDQ, MLDQ, MU, ML, DQRELY, IER)
215  * The arguments are:
216  * NLOCAL = local size of vectors
217  * MUDQ,MLDQ = upper and lower half-bandwidths to be used in the computation
218  * of the local Jacobian blocks by difference quotients.
219  * These may be smaller than the true half-bandwidths of the
220  * Jacobian of the local block of g, when smaller values may
221  * provide greater efficiency.
222  * MU, ML = upper and lower half-bandwidths of the band matrix that
223  * is retained as an approximation of the local Jacobian block.
224  * These may be smaller than MUDQ and MLDQ.
225  * DQRELY = relative increment factor in y for difference quotients
226  * (optional). 0.0 indicates the default, sqrt(UNIT_ROUNDOFF).
227  * IER = return completion flag: IER=0: success, IER<0: an error occured
228  *
229  * (4.5) To specify whether the linear solver should use the supplied FIDAJTIMES or the
230  * internal finite difference approximation, make the call
231  * CALL FIDASPILSSETJAC(FLAG, IER)
232  * where FLAG=0 for finite differences approxaimtion or
233  * FLAG=1 to use the supplied routine FIDAJTIMES
234  *
235  * (5) Re-initialization: FIDAREINIT, FIDABBDREINIT
236  * If a sequence of problems of the same size is being solved using the SPGMR or
237  * SPBCG linear solver in combination with the IDABBDPRE preconditioner, then the
238  * IDA package can be reinitialized for the second and subsequent problems
239  * so as to avoid further memory allocation. First, in place of the call
240  * to FIDAMALLOC, make the following call:
241  * CALL FIDAREINIT(T0, Y0, YP0, IATOL, RTOL, ATOL, ID, CONSTR, IER)
242  * The arguments have the same names and meanings as those of FIDAMALLOC.
243  * FIDAREINIT performs the same initializations as FIDAMALLOC, but does no
244  * memory allocation for IDA data structures, using instead the existing
245  * internal memory created by the previous FIDAMALLOC call. Following the call
246  * to FIDAREINIT, a call to FIDABBDINIT may or may not be needed. If the input
247  * arguments are the same, no FIDABBDINIT call is needed. If there is a change
248  * in input arguments other than MU, ML or MAXL, then the user program should call
249  * FIDABBDREINIT. The arguments of the FIDABBDREINIT routine have the
250  * same names and meanings as FIDABBDINIT. Finally, if the value of MU, ML, or
251  * MAXL is being changed, then a call to FIDABBDINIT must be made.
252  *
253  * (6) The solver: FIDASOLVE
254  * To solve the DAE system, make the following call:
255  * CALL FIDASOLVE (TOUT, TRET, Y, YP, ITASK, IER)
256  * The arguments are:
257  * TOUT = next value of t at which a solution is desired (input)
258  * TRET = value of t reached by the solver on output
259  * Y = array containing the computed solution on output
260  * YP = array containing current value of y'
261  * ITASK = task indicator: 1 = normal mode (overshoot TOUT and interpolate)
262  * 2 = one-step mode (return after each internal step taken)
263  * 3 = normal tstop mode (like 1, but integration never proceeds past
264  * TSTOP, which must be specified through a call to FIDASETRIN
265  * using the key 'STOP_TIME'
266  * 4 = one step tstop (like 2, but integration never goes past TSTOP)
267  * IER = completion flag: 0 = success, 1 = tstop return, 2 = root return,
268  * values -1 ... -10 are various failure modes (see IDA manual).
269  * The current values of the optional outputs are available in IOUT and ROUT.
270  *
271  * (7) Optional outputs: FIDABBDOPT
272  * Optional outputs specific to the SPGMR/SPBCG/SPTFQMR solver are available
273  * in IOUT(13)...IOUT(21)
274  *
275  * To obtain the optional outputs associated with the IDABBDPRE module, make
276  * the following call:
277  * CALL FIDABBDOPT (LENRWBBD, LENIWBBD, NGEBBD)
278  * The arguments returned are:
279  * LENRWBBD = length of real preconditioner work space, in realtype words.
280  * This size is local to the current process.
281  * LENIWBBD = length of integer preconditioner work space, in integer words.
282  * This size is local to the current process.
283  * NGEBBD = number of G(t,y,y') evaluations (calls to FIDAGLOCFN) so far.
284  *
285  * (8) Memory freeing: FIDAFREE
286  * To the free the internal memory created by the calls to FNVINITP and
287  * FIDAMALLOC, make the following call:
288  * CALL FIDAFREE
289  *
290  * ==============================================================================
291  */
