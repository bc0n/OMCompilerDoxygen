58 /*-----------------------------------------------------------------
59  Types : ARKSlsSparseJacFn
60 
61  A sparse Jacobian approximation function Jac must be of type
62  ARKSlsSparseJacFn. Its parameters are:
63 
64  t is the current value of the independent variable t.
65 
66  y is the current value of the dependent variable vector,
67  namely the predicted value of y(t)
68 
69  fy is the vector f(t,y).
70  namely the predicted value of y'(t)
71 
72  JacMat is the compressed sparse column matrix (of type SlsMat)
73  to be loaded by an ARKSlsSparseJacFn routine with an
74  approximation to the system Jacobian matrix
75  J = (df_i/dy_j) at the point (t,y).
76  Note that JacMat is NOT preset to zero!
77  Matrix data is for the nonzero entries of the Jacobian stored
78  in compressed column format. Row indices of entries in
79  column j are stored in JacMat->rowvals[colptrs[j]]
80  through JacMat->rowvals[colptrs[j+i]-1]
81  and corresponding numerical values of the Jacobian are stored
82  in the same entries of JacMat->data.
83 
84  J_data is a pointer to user Jacobian data - the same as the
85  user_data parameter passed to ARKodeSetFdata.
86 
87  tmp1, tmp2, tmp3 are pointers to memory allocated for
88  N_Vectors which can be used by an ARKSparseJacFn routine
89  as temporary storage or work space.
90 
91  A ARKSlsSparseJacFn should return 0 if successful, a positive
92  value if a recoverable error occurred, and a negative value if
93  a nonrecoverable error occurred.
94 
95  NOTE: If the user's Jacobian routine needs other quantities,
96  they are accessible as follows: hcur (the current stepsize)
97  and ewt (the error weight vector) are accessible through
98  ARKodeGetCurrentStep and ARKodeGetErrWeights, respectively
99  (see arkode.h). The unit roundoff is available as
100  UNIT_ROUNDOFF defined in sundials_types.h.
101 
102 ---------------------------------------------------------------*/
103 typedef int (*ARKSlsSparseJacFn)(realtype t,
104  N_Vector y, N_Vector fy,
105  SlsMat JacMat, void *user_data,
106  N_Vector tmp1, N_Vector tmp2,
107  N_Vector tmp3);

109 /*-----------------------------------------------------------------
110  Types : ARKSlsSparseMassFn
111 
112  A sparse mass matrix approximation function Mass must be of type
113  ARKSlsSparseMassFn. Its parameters are:
114 
115  t is the current value of the independent variable t.
116 
117  MassMat is the compressed sparse column matrix (of type SlsMat)
118  to be loaded by an ARKSlsSparseJacFn routine with an
119  approximation to the system mass matrix. Note that MassMat
120  is NOT preset to zero! Matrix data is for the nonzero entries
121  of the mass matrix stored in compressed column format. Row
122  indices of entries in column j are stored in
123  MassMat->rowvals[colptrs[j]] through
124  MassMat->rowvals[colptrs[j+i]-1]
125  and corresponding numerical values of the mass matrix are
126  stored in the same entries of MassMat->data.
127 
128  M_data is a pointer to user mass matrix data - the same as the
129  user_data parameter passed to ARKodeSetFdata.
130 
131  tmp1, tmp2, tmp3 are pointers to memory allocated for
132  N_Vectors which can be used by an ARKSparseJacFn routine
133  as temporary storage or work space.
134 
135  A ARKSlsSparseMassFn should return 0 if successful, a positive
136  value if a recoverable error occurred, and a negative int if a
137  nonrecoverable error occurred.
138 
139 ---------------------------------------------------------------*/
140 typedef int (*ARKSlsSparseMassFn)(realtype t, SlsMat MassMat,
