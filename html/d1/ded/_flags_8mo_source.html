32 encapsulated package Flags
33 " file: Flags.mo
34  package: Flags
35  description: Tools for using compiler flags.
36 
37  This package contains function for using compiler flags. There are two types
38  of flags, debug flag and configuration flags. The flags are stored and
39  retrieved with set/getGlobalRoot so that they can be accessed everywhere in
40  the compiler.
41 
42  Configuration flags are flags such as --std which affects the behaviour of the
43  compiler. These flags can have different types, see the FlagData uniontype
44  below, and they also have a default value. There is also another package,
45  Config, which acts as a wrapper for many of these flags.
46 
47  Debug flags are boolean flags specified with -d, which can be used together
48  with the Debug package. They are typically used to enable printing of extra
49  information that helps debugging, such as the failtrace flag. Unlike
50  configuration flags they are only on or off, i.e. true or false.
51 
52  To add a new flag, simply add a new constant of either DebugFlag or ConfigFlag
53  type below, and then add it to either the allDebugFlags or allConfigFlags list
54  depending on which type it is.
55  "
