74 /*
75  * -----------------------------------------------------------------
76  * Types : CVSlsSparseJacFn
77  * -----------------------------------------------------------------
78  *
79  * A sparse Jacobian approximation function jac must be of type
80  * CVSlsSparseJacFn.
81  * Its parameters are:
82  *
83  * t is the current value of the independent variable t.
84  *
85  * y is the current value of the dependent variable vector,
86  * namely the predicted value of y(t).
87  *
88  * fy is the vector f(t,y).
89  * namely the predicted value of y'(t).
90  *
91  * JacMat is the compressed sparse column matrix (of type SlsMat)
92  * to be loaded by an CVSlsSparseJacFn routine with an approximation
93  * to the system Jacobian matrix
94  * J = J = (df_i/dy_j) at the point (t,y).
95  * Note that JacMat is NOT preset to zero!
96  * Matrix data is for the nonzero entries of the Jacobian stored in
97  * compressed column format. Row indices of entries in
98  * column j are stored in J->rowvals[colptrs[j]]
99  * through J->rowvals[colptrs[j+i]-1]
100  * and corresponding numerical values of the Jacobian are stored
101  * in the same entries of J->data.
102  *
103  * J_data is a pointer to user Jacobian data - the same as the
104  * user_data parameter passed to CVodeSetFdata.
105  *
106  * tmp1, tmp2, tmp3 are pointers to memory allocated for
107  * N_Vectors which can be used by an CVSparseJacFn routine
108  * as temporary storage or work space.
109  *
110  * A CVSlsSparseJacFn should return
111  * 0 if successful,
112  * a positive int if a recoverable error occurred, or
113  * a negative int if a nonrecoverable error occurred.
114  *
115  * -----------------------------------------------------------------
116  *
117  * NOTE: If the user's Jacobian routine needs other quantities,
118  * they are accessible as follows: hcur (the current stepsize)
119  * and ewt (the error weight vector) are accessible through
120  * CVodeGetCurrentStep and CVodeGetErrWeights, respectively
121  * (see cvode.h). The unit roundoff is available as
122  * UNIT_ROUNDOFF defined in sundials_types.h.
123  *
124  * -----------------------------------------------------------------
125  */
126
