18  * This is the interface file for the main ARKODE integrator.
19  *---------------------------------------------------------------
20  * ARKODE is used to solve numerically the ordinary initial value
21  * problem:
22  * M(t)*y' = f(t,y),
23  * y(t0) = y0,
24  * where t0, y0 in R^N, M(t)*y: R x R^N -> R^N, and
25  * f: R x R^N -> R^N are given.

42 /*---------------------------------------------------------------
43  Enumerations for inputs to ARKodeCreate and ARKode.
44 -----------------------------------------------------------------
45  Symbolic constants for the iter parameter to
46  ARKodeCreate and the input parameter itask to ARKode, are given
47  below.
48 
49  itask: The itask input parameter to ARKode indicates the job
50  of the solver for the next user step. The ARK_NORMAL
51  itask is to have the solver take internal steps until
52  it has reached or just passed the user specified tout
53  parameter. The solver then interpolates in order to
54  return an approximate value of y(tout). The ARK_ONE_STEP
55  option tells the solver to just take one internal step
56  and return the solution at the point reached by that
57  step. If fixed time steps are desired, then the user
58  can run in ARK_ONE_STEP mode, and set hmin and hmax to
59  be the desired time step -- this will effectively
60  disable all temporal error control, and the user may
61  need to increase the iteration counts for the nonlinear
62  and linear solvers to guarantee convergence for larger
63  step sizes.
64 ---------------------------------------------------------------*/

116 /*---------------------------------------------------------------
117  Type : ARKRhsFn
118 -----------------------------------------------------------------
119  The f functions which define the right hand side of the ODE
120  system M*y' = fe(t,y) + fi(t,y) must have type ARKRhsFn.
121  f takes as input the independent variable value t, and the
122  dependent variable vector y. It stores the result of fe(t,y)
123  or fi(t,y) in the vector ydot. The y and ydot arguments are of
124  type N_Vector.
125  (Allocation of memory for ydot is handled within ARKODE)
126  The user_data parameter is the same as the user_data
127  parameter set by the user through the ARKodeSetUserData routine.
128  This user-supplied pointer is passed to the user's fe or fi
129  function every time it is called.
130 
131  A ARKRhsFn should return 0 if successful, a negative value if
132  an unrecoverable error occured, and a positive value if a
133  recoverable error (e.g. invalid y values) occured.
134  If an unrecoverable occured, the integration is halted.
135  If a recoverable error occured, then (in most cases) ARKODE
136  will try to correct and retry.
137 ---------------------------------------------------------------*/
138 typedef int (*ARKRhsFn)(realtype t, N_Vector y,
139  N_Vector ydot, void *user_data);
140 
141 /*---------------------------------------------------------------
142  Type : ARKRootFn
143 -----------------------------------------------------------------
144  A function g, which defines a set of functions g_i(t,y) whose
145  roots are sought during the integration, must have type
146  ARKRootFn. The function g takes as input the independent
147  variable value t, and the dependent variable vector y. It
148  stores the nrtfn values g_i(t,y) in the realtype array gout.
149  (Allocation of memory for gout is handled within ARKODE.)
150  The user_data parameter is the same as that passed by the user
151  to the ARKodeSetUserData routine. This user-supplied pointer
152  is passed to the user's g function every time it is called.
153 
154  A ARKRootFn should return 0 if successful or a non-zero value
155  if an error occured (in which case the integration will be
156  halted).
157 ---------------------------------------------------------------*/
158 typedef int (*ARKRootFn)(realtype t, N_Vector y,
159  realtype *gout, void *user_data);
160 
161 /*---------------------------------------------------------------
162  Type : ARKEwtFn
163 -----------------------------------------------------------------
164  A function e, which sets the error weight vector ewt, must have
165  type ARKEwtFn. The function e takes as input the current
166  dependent variable y. It must set the vector of error weights
167  used in the WRMS norm:
168 
169  ||y||_WRMS = sqrt [ 1/N * sum ( ewt_i * y_i)^2 ]
170 
171  Typically, the vector ewt has components:
172 
173  ewt_i = 1 / (reltol * |y_i| + abstol_i)
174 
175  The user_data parameter is the same as that passed by the user
176  to the ARKodeSetUserData routine. This user-supplied pointer
177  is passed to the user's e function every time it is called.
178  A ARKEwtFn e must return 0 if the error weight vector has been
179  successfuly set and a non-zero value otherwise.
180 ---------------------------------------------------------------*/
181 typedef int (*ARKEwtFn)(N_Vector y, N_Vector ewt, void *user_data);
182 
183 /*---------------------------------------------------------------
184  Type : ARKRwtFn
185 -----------------------------------------------------------------
186  A function r, which sets the residual weight vector rwt, must
187  have type ARKRwtFn. The function r takes as input the current
188  dependent variable y. It must set the vector of residual
189  weights used in the WRMS norm:
190 
191  ||v||_WRMS = sqrt [ 1/N * sum ( rwt_i * v_i)^2 ]
192 
193  Typically, the vector rwt has components:
194 
195  rwt_i = 1 / (reltol * |(M*y)_i| + rabstol_i)
196 
197  The user_data parameter is the same as that passed by the user
198  to the ARKodeSetUserData routine. This user-supplied pointer
199  is passed to the user's r function every time it is called.
200  A ARKRwtFn e must return 0 if the residual weight vector has
201  been successfuly set and a non-zero value otherwise.
202 ---------------------------------------------------------------*/
203 typedef int (*ARKRwtFn)(N_Vector y, N_Vector rwt, void *user_data);
204 
205 /*---------------------------------------------------------------
206  Type : ARKErrHandlerFn
207 -----------------------------------------------------------------
208  A function eh, which handles error messages, must have type
209  ARKErrHandlerFn. The function eh takes as input the error code,
210  the name of the module reporting the error, the error message,
211  and a pointer to user data, the same as that passed to
212  ARKodeSetUserData.
213 
214  All error codes are negative, except ARK_WARNING which indicates
215  a warning (the solver continues).
216 
217  A ARKErrHandlerFn has no return value.

219 typedef void (*ARKErrHandlerFn)(int error_code, const char *module,
220  const char *function, char *msg,
221  void *user_data);

223 /*---------------------------------------------------------------
224  Type : ARKAdaptFn
225 -----------------------------------------------------------------
226  A function which sets the new time step h, must have type
227  ARKAdaptFn. The function takes as input the current dependent
228  variable y, the current time t, the last 3 step sizes h,
229  the last 3 error estimates, the method order q, the embedding
230  order p, and a pointer to user data. The function must set the
231  scalar step size for the upcoming time step. This value will
232  subsequently be bounded by the user-supplied values for the
233  minimum and maximum allowed time step, and the time step
234  satisfying the explicit stability restriction. The user_data
235  parameter is the same as that passed by the user to the
236  ARKodeSetUserData routine. This user-supplied pointer is passed
237  to the function every time it is called.
238 
239  A ARKAdaptFn must return 0 if the new time step has been
240  successfuly set and a non-zero value otherwise.
241 ---------------------------------------------------------------*/
242 typedef int (*ARKAdaptFn)(N_Vector y, realtype t, realtype h1,
243  realtype h2, realtype h3,
244  realtype e1, realtype e2,
245  realtype e3, int q, int p,
246  realtype *hnew, void *user_data);
247 
248 /*---------------------------------------------------------------
249  Type : ARKExpStabFn
250 -----------------------------------------------------------------
251  A function which returns the time step satisfying the stability
252  restriction for the explicit portion of the ODE. The function
253  takes as input the current dependent variable y, the current
254  time t, and a pointer to user data. The function must set the
255  scalar step size satisfying the stability restriction for the
256  upcoming time step. This value will subsequently be bounded by
257  the user-supplied values for the minimum and maximum allowed
258  time step, and the accuracy-based time step. The user_data
259  parameter is the same as that passed by the user to the
260  ARKodeSetUserData routine. This user-supplied pointer is passed
261  to the function every time it is called.
262 
263  If this function is not supplied (NULL), or if it returns a
264  negative time step size, then ARKode will assume that there is
265  no explicit stability restriction on the time step size.
266 
267  A ARKExpStabFn must return 0 if the step size limit has been
268  successfuly set and a non-zero value otherwise.
269 ---------------------------------------------------------------*/
270 typedef int (*ARKExpStabFn)(N_Vector y, realtype t,
271  realtype *hstab, void *user_data);

273 /*---------------------------------------------------------------
274  Type : ARKVecResizeFn
275 -----------------------------------------------------------------
276  When calling ARKodeResize, the user may specify a vector resize
277  function to be used to convert any existing N_Vectors in the
278  ARKode memory structure to the new problem size. This would
279  typically be used if there is a user-supplied N_Vector module
280  that allows dynamic resizing of the vector data structures
281  without the need to delete/allocate memory on each call.
282 
283  The default behavior will be to delete the vector memory and
284  re-clone from the new vector; if this is the desired behavior
285  then specification of the ARKVecResizeFn is not recommended.
286 
287  The first argument, 'y', is the vector to be resized.
288 
289  The second argument, 'ytemplate', is the user-provided vector
290  with the "new" size, that may be used as a template.
291 
292  The third argument, 'user_data', is a user-provided data
293  structure to ARKodeResize, in case additional data is
294  necessary for the resize operation.
295 
296  A ARKVecResizeFn should return 0 if successful, and a nonzero
297  value if an error occurred.
298 ---------------------------------------------------------------*/
299 typedef int (*ARKVecResizeFn)(N_Vector y, N_Vector ytemplate,
300  void *user_data);

943 /*---------------------------------------------------------------
944  Function : ARKode
945 -----------------------------------------------------------------
946  ARKode integrates the ODE over an interval in t.
947 
948  ARKode may be run in one of two modes (ARK_NORMAL or
949  ARK_ONE_STEP), as determined by the itask argument:
950 
951  If itask is ARK_NORMAL, then the solver integrates from its
952  current internal t value to a point at or beyond tout, then
953  interpolates to t = tout and returns y(tout) in the user-
954  allocated vector yout. This interpolation is typically less
955  accurate than the full time step solutions produced by the
956  solver, since the interpolating polynomial relies on the
957  internal stage solutions, that may have reduced accuracy in
958  comparison with the full time step solutions. If the user
959  wishes that this returned value have full method accuracy, they
960  may issue a call to ARKodeSetStopTime before the call to ARKode
961  to specify a fixed stop time to end the time step and return to
962  the user. Once the integrator returns at a tstop time, any
963  future testing for tstop is disabled (and can be reenabled only
964  though a new call to ARKodeSetStopTime).
965 
966  If itask is ARK_ONE_STEP, then the solver takes one internal
967  time step and returns in yout the value of y at the new internal
968  time. In this case, tout is used only during the first call to
969  ARKode to determine the direction of integration and the rough
970  scale of the t variable. As with the ARK_NORMAL mode, a user
971  may specify a specific stop time for output of this step,
972  assuming that the requested step is smaller than the step taken
973  by the method.
974 
975  The time reached by the solver is placed in (*tret). The
976  user is responsible for allocating the memory for this value.
977 
978  arkode_mem is the pointer to ARKODE memory returned by
979  ARKodeCreate.
980 
981  tout is the next time at which a computed solution is desired.
982 
983  yout is the computed solution vector. In ARK_NORMAL mode with no
984  errors and no roots found, yout=y(tout).
985 
986  tret is a pointer to a real location. ARKode sets (*tret) to
987  the time reached by the solver and returns
988  yout=y(*tret).
989 
990  itask is ARK_NORMAL or ARK_ONE_STEP, as described above.
991 
992  Here is a brief description of each return value:
993 
994  ARK_SUCCESS: ARKode succeeded and no roots were found.
995 
996  ARK_ROOT_RETURN: ARKode succeeded, and found one or more roots.
997  If nrtfn > 1, call ARKodeGetRootInfo to see
998  which g_i were found to have a root at (*tret).
999 
1000  ARK_TSTOP_RETURN: ARKode succeeded and returned at tstop.
1001 
1002  ARK_MEM_NULL: The arkode_mem argument was NULL.
1003 
1004  ARK_NO_MALLOC: arkode_mem was not allocated.
1005 
1006  ARK_ILL_INPUT: One of the inputs to ARKode is illegal. This
1007  includes the situation when a component of the
1008  error weight vectors becomes < 0 during
1009  internal time-stepping. It also includes the
1010  situation where a root of one of the root
1011  functions was found both at t0 and very near t0.
1012  The ILL_INPUT flag will also be returned if the
1013  linear solver routine ARK--- (called by the user
1014  after calling ARKodeCreate) failed to set one of
1015  the linear solver-related fields in arkode_mem or
1016  if the linear solver's init routine failed. In
1017  any case, the user should see the printed
1018  error message for more details.
1019 
1020  ARK_TOO_MUCH_WORK: The solver took mxstep internal steps but
1021  could not reach tout prior to the maximum number
1022  of steps (ark_mxstep).
1023 
1024  ARK_TOO_MUCH_ACC: The solver could not satisfy the accuracy
1025  demanded by the user for some internal step.
1026 
1027  ARK_ERR_FAILURE: Error test failures occurred too many times
1028  (= ark_maxnef) during one internal time step
1029  or occurred with |h| = hmin.
1030 
1031  ARK_CONV_FAILURE: Convergence test failures occurred too many
1032  times (= ark_maxncf) during one internal time
1033  step or occurred with |h| = hmin.
1034 
1035  ARK_LINIT_FAIL: The linear solver's initialization function
1036  failed.
1037 
1038  ARK_LSETUP_FAIL: The linear solver's setup routine failed in an
1039  unrecoverable manner.
1040 
1041  ARK_LSOLVE_FAIL: The linear solver's solve routine failed in an
1042  unrecoverable manner.
1043 ---------------------------------------------------------------*/
1044 SUNDIALS_EXPORT int ARKode(void *arkode_mem, realtype tout,
