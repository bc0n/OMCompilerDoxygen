42 // Purpose
43 // The example tests the successive right-hand sides capabilities of ML
44 // and Belos on a heat flow u_t = u_xx problem.
45 //
46 // A sequence of linear systems with the same coefficient matrix and
47 // different right-hand sides is solved. A seed space is generated dynamically,
48 // and a deflated linear system is solved. After each solves, the first
49 // few Krylov vectors are saved, and used to reduce the number of iterations
50 // for later solves.
51 // The optimal numbers of vectors to deflate and save are not known.
52 // Presently, the maximum number of vectors to deflate (seed space dimension)
53 // and to save are user paraemters.
54 // The seed space dimension is less than or equal to total number of vectors saved.
55 // The difference between the seed space dimension and the total number of vectors,
56 // is the number of vectors used to update the seed space after each solve.
57 // I guess that a seed space whose dimension is a small fraction of the total space
58 // will be best.
59 //
60 // maxSave=1 and maxDeflate=0 uses no recycling (not tested ).
61 //
62 // TODO: Instrument with timers, so that we can tell what is going on besides
63 // by counting the numbers of iterations.
64 //
65 //
66 // \author David M. Day
67 //
68 // \data Last modified 2007 December 11
