45 /* -------------------------------------------------------------------- */
46 /* 'CommInfoOP' records communication information: no. of neighboring */
47 /* processors, an array of pointers to the neighbors, and 'add_rcfd' */
48 /* which indicates whether or not received information is added to */
49 /* existing values or overwrites existing values. That is, if we */
50 /* receive a value that should be stored in the kth location of a */
51 /* vector, do we add it with the value already in the kth location or */
52 /* do we over-write. Additionally, if remap != NULL, the locations of */
53 /* the values are effectively permuted after communication. In */
54 /* particular, remap[i] = -1 indicates that the ith value of the vector */
55 /* is no longer used while remap[i] = k indicates that the ith value of */
56 /* the vector is actually stored in the kth index of the array. */
57 /* -------------------------------------------------------------------- */
58
