24  * ==============================================================================
25  *
26  * FCVBP Interface Package
27  *
28  * The FCVBP Interface Package is a package of C functions which,
29  * together with the FCVODE Interface Package, support the use of the
30  * CVODE solver (serial version) with the CVBANDPRE preconditioner module,
31  * for the solution of ODE systems in a mixed Fortran/C setting. The
32  * combination of CVODE and CVBANDPRE solves systems dy/dt = f(t,y) with the
33  * SPGMR (scaled preconditioned GMRES), SPTFQMR (scaled preconditioned TFQMR),
34  * or SPBCG (scaled preconditioned Bi-CGSTAB) method for the linear systems
35  * that arise, and with a banded difference quotient Jacobian-based preconditioner.
36  *
37  * The user-callable functions in this package, with the corresponding
38  * CVODE and CVBBDPRE functions, are as follows:
39  * FCVBPINIT interfaces to CVBandPrecInit
40  * FCVBPOPT accesses optional outputs
41  *
42  * In addition to the Fortran right-hand side function FCVFUN, the
43  * user may (optionally) supply a routine FCVJTIMES which is called by
44  * the interface function FCVJtimes of type CVSpilsJtimesFn.
45  * (The names of all user-supplied routines here are fixed, in order to
46  * maximize portability for the resulting mixed-language program.)
47  *
48  * Important note on portability.
49  * In this package, the names of the interface functions, and the names of
50  * the Fortran user routines called by them, appear as dummy names
51  * which are mapped to actual values by a series of definitions in the
52  * header file fcvbp.h.
53  *
54  * ==============================================================================
55  *
56  * Usage of the FCVODE/FCVBP Interface Packages
57  *
58  * The usage of the combined interface packages FCVODE and FCVBP requires
59  * calls to seven to ten interface functions, and one or two user-supplied
60  * routines which define the problem to be solved and indirectly define
61  * the preconditioner. These function calls and user routines are
62  * summarized separately below.
63  *
64  * Some details are omitted, and the user is referred to the CVODE user document
65  * for more complete information.
66  *
67  * (1) User-supplied right-hand side routine: FCVFUN
68  * The user must in all cases supply the following Fortran routine
69  * SUBROUTINE FCVFUN (T, Y, YDOT, IPAR, RPAR, IER)
70  * DIMENSION Y(*), YDOT(*), IPAR(*), RPAR(*)
71  * It must set the YDOT array to f(t,y), the right-hand side of the ODE
72  * system, as function of T = t and the array Y = y. Here Y and YDOT
73  * are distributed vectors.
74  *
75  * (2) Optional user-supplied Jacobian-vector product routine: FCVJTIMES
76  * As an option, the user may supply a routine that computes the product
77  * of the system Jacobian J = df/dy and a given vector v. If supplied, it
78  * must have the following form:
79  * SUBROUTINE FCVJTIMES (V, FJV, T, Y, FY, EWT, IPAR, RPAR, WORK, IER)
80  * DIMENSION V(*), FJV(*), Y(*), FY(*), EWT(*), IPAR(*), RPAR(*), WORK(*)
81  * Typically this routine will use only NEQ, T, Y, V, and FJV. It must
82  * compute the product vector Jv, where the vector v is stored in V, and store
83  * the product in FJV. On return, set IER = 0 if FCVJTIMES was successful,
84  * and nonzero otherwise.
85  *
86  * (3) Initialization: FNVINITS, FCVMALLOC, FCVBPINIT.
87  *
88  * (3.1) To initialize the serial vector specification, the user must make
89  * the following call:
90  * CALL FNVINITS(NEQ, IER)
91  * where NEQ is the problem size and IER is a return completion flag.
92  * Possible values for IER are 0 = success, -1 = failure.
93  *
94  * (3.2) To set various problem and solution parameters and allocate
95  * internal memory for CVODE, make the following call:
96  * CALL FCVMALLOC(T0, Y0, METH, ITMETH, IATOL, RTOL, ATOL,
97  * 1 IOUT, ROUT, IPAR, RPAR, IER)
98  * The arguments are:
99  * T0 = initial value of t
100  * Y0 = array of initial conditions
101  * METH = basic integration method: 1 = Adams (nonstiff), 2 = BDF (stiff)
102  * ITMETH = nonlinear iteration method: 1 = functional iteration, 2 = Newton iter.
103  * IATOL = type for absolute tolerance ATOL: 1 = scalar, 2 = array
104  * RTOL = relative tolerance (scalar)
105  * ATOL = absolute tolerance (scalar or array)
106  * IOUT = array of length 21 for integer optional outputs
107  * (declare as INTEGER*4 or INTEGER*8 according to C type long int)
108  * ROUT = array of length 6 for real optional outputs
109  * IPAR = array with user integer data
110  * (declare as INTEGER*4 or INTEGER*8 according to C type long int)
111  * RPAR = array with user real data
112  * IER = return completion flag. Values are 0 = success, and -1 = failure.
113  * See printed message for details in case of failure.
114  *
115  * (3.3) To allocate memory and initialize data associated with the CVBANDPRE
116  * preconditioner, make the following call:
117  * CALL FCVBPINIT(NEQ, MU, ML, IER)
118  * The arguments are:
119  * NEQ = problem size
120  * MU, ML = upper and lower half-bandwidths of the band matrix that
121  * is retained as an approximation of the Jacobian.
122  * IER = return completion flag: IER=0: success, IER<0: and error occurred
123  *
124  * (3.4A) To specify the SPGMR linear solver with the CVBANDPRE preconditioner,
125  * make the following call
126  * CALL FCVSPGMR(IPRETYPE, IGSTYPE, MAXL, DELT, IER)
127  * The arguments are:
128  * IPRETYPE = preconditioner type:
129  * 0 = none
130  * 1 = left only
131  * 2 = right only
132  * 3 = both sides.
133  * IGSTYPE = Gram-schmidt process type: 0 = modified G-S, 1 = classical G-S.
134  * MAXL = maximum Krylov subspace dimension; 0 indicates default.
135  * DELT = linear convergence tolerance factor; 0.0 indicates default.
136  * IER = return completion flag: IER=0: success, IER<0: ans error occurred
137  *
138  * (3.4B) To specify the SPBCG linear solver with the CVBANDPRE preconditioner,
139  * make the following call
140  * CALL FCVSPBCG(IPRETYPE, MAXL, DELT, IER)
141  * The arguments are:
142  * IPRETYPE = preconditioner type:
143  * 0 = none
144  * 1 = left only
145  * 2 = right only
146  * 3 = both sides.
147  * MAXL = maximum Krylov subspace dimension; 0 indicates default.
148  * DELT = linear convergence tolerance factor; 0.0 indicates default.
149  * IER = return completion flag: IER=0: success, IER<0: ans error occurred
150  *
151  * (3.4C) To specify the SPTFQMR linear solver with the CVBANDPRE preconditioner,
152  * make the following call
153  * CALL FCVSPTFQMR(IPRETYPE, MAXL, DELT, IER)
154  * The arguments are:
155  * IPRETYPE = preconditioner type:
156  * 0 = none
157  * 1 = left only
158  * 2 = right only
159  * 3 = both sides.
160  * MAXL = maximum Krylov subspace dimension; 0 indicates default.
161  * DELT = linear convergence tolerance factor; 0.0 indicates default.
162  * IER = return completion flag: IER=0: success, IER<0: ans error occurred
163  *
164  * (3.5) To specify whether the Krylov linear solver (GMRES, Bi-CGSTAB, or TFQMR)
165  * should use the supplied FCVJTIMES or the internal finite difference approximation,
166  * make the call
167  * CALL FCVSPILSSETJAC(FLAG, IER)
168  * where FLAG=0 for finite differences approxaimtion or
169  * FLAG=1 to use the supplied routine FCVJTIMES
170  *
171  * (4) The integrator: FCVODE
172  * Carrying out the integration is accomplished by making calls as follows:
173  * CALL FCVODE (TOUT, T, Y, ITASK, IER)
174  * The arguments are:
175  * TOUT = next value of t at which a solution is desired (input)
176  * T = value of t reached by the solver on output
177  * Y = array containing the computed solution on output
178  * ITASK = task indicator: 1 = normal mode (overshoot TOUT and interpolate);
179  * 2 = one-step mode (return after each internal step taken);
180  * 3 = normal mode with TSTOP; 4 = one-step mode with TSTOP.
181  * IER = completion flag: 0 = success, 1 = TSTOP return, 2 = root return,
182  * negative values are various failure modes (see CVODE User Guide).
183  * The current values of the optional outputs are available in IOUT and ROUT.
184  *
185  * (5) Optional outputs: FCVBPOPT
186  * Optional outputs specific to the SP* solver are LRW, LIW, LFLG, NFELS, NJTV,
187  * NPE, NPS, NLI, NCFL, stored in IOUT(13)...IOUT(21).
188  * To obtain the optional outputs associated with the CVBANDPRE module, make
189  * the following call:
190  * CALL FCVBPOPT(LENRWBP, LENIWBP, NFEBP)
191  * The arguments returned are:
192  * LENRWBP = length of real preconditioner work space, in realtype words.
193  * This size is local to the current processor.
194  * LENIWBP = length of integer preconditioner work space, in integer words.
195  * This size is local to the current processor.
196  * NFEBP = number of f(t,y) evaluations for CVBANDPRE
197  *
198  * (6) Computing solution derivatives: FCVDKY
199  * To obtain a derivative of the solution (optionally), of order up to
200  * the current method order, make the following call:
201  * CALL FCVDKY (T, K, DKY)
202  * The arguments are:
203  * T = value of t at which solution derivative is desired
204  * K = derivative order (0 .le. K .le. QU)
205  * DKY = array containing computed K-th derivative of y on return
206  *
207  * (7) Memory freeing: FCVFREE
208  * To the free the internal memory created by the calls to FNVINITS,
209  * FCVMALLOC, and FCVBPINIT, make the following call:
210  * CALL FCVFREE
211  *
212  * ==============================================================================
213  */
