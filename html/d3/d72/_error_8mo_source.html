33 "
34  file: Error.mo
35  package: Error
36  description: Error handling
37 
38 
39  This file contains the Error handling for the Compiler. The following steps
40  are used to add a new error message:
41 
42  1) Add a new ErrorID constant below with an unique id.
43 
44  2) Add a new entry in the errorTable. Each entry is a tuple consisting of
45  the ErrorID, a MessageType, a Severity and a message string. See the
46  MessageType and Severity uniontypes below for more information about
47  them.
48 
49  The message string is the error message that should be displayed, which
50  may contain directives to insert tokens given when the message is used.
51  These directives are:
52 
53  %s: Inserts the next token in the list.
54  %n: Inserts token number n in the list, where n is a number from 1 to 9.
55 
56  Note that these two directives do not affect each other. I.e. %s will move
57  to the next token in the list regardless of any positional directives, and
58  %1 will always point to the first token regardless of any %s before it.
59  An example:
60 
61  Message: '%2: This is a %s of %2 %s and %1'
62  Tokens: {'test', 'error', 'directives', 'messages'}
63  Result: 'error: This is a test of error messages and directives'
64 
65  3) Use the new error message by calling addSourceMessage or addMessage with
66  it's ErrorID.
