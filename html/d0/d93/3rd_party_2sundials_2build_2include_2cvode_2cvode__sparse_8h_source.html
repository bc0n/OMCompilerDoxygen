67 /*
68  * -----------------------------------------------------------------
69  * Types : CVSlsSparseJacFn
70  * -----------------------------------------------------------------
71  *
72  * A sparse Jacobian approximation function jac must be of type
73  * CVSlsSparseJacFn.
74  * Its parameters are:
75  *
76  * t is the current value of the independent variable t.
77  *
78  * y is the current value of the dependent variable vector,
79  * namely the predicted value of y(t).
80  *
81  * fy is the vector f(t,y).
82  * namely the predicted value of y'(t).
83  *
84  * JacMat is the compressed sparse column matrix (of type SlsMat)
85  * to be loaded by an CVSlsSparseJacFn routine with an approximation
86  * to the system Jacobian matrix
87  * J = J = (df_i/dy_j) at the point (t,y).
88  * Note that JacMat is NOT preset to zero!
89  * Matrix data is for the nonzero entries of the Jacobian stored in
90  * compressed column format. Row indices of entries in
91  * column j are stored in J->rowvals[colptrs[j]]
92  * through J->rowvals[colptrs[j+i]-1]
93  * and corresponding numerical values of the Jacobian are stored
94  * in the same entries of J->data.
95  *
96  * J_data is a pointer to user Jacobian data - the same as the
97  * user_data parameter passed to CVodeSetFdata.
98  *
99  * tmp1, tmp2, tmp3 are pointers to memory allocated for
100  * N_Vectors which can be used by an CVSparseJacFn routine
101  * as temporary storage or work space.
102  *
103  * A CVSlsSparseJacFn should return
104  * 0 if successful,
105  * a positive int if a recoverable error occurred, or
106  * a negative int if a nonrecoverable error occurred.
107  *
108  * -----------------------------------------------------------------
109  *
110  * NOTE: If the user's Jacobian routine needs other quantities,
111  * they are accessible as follows: hcur (the current stepsize)
112  * and ewt (the error weight vector) are accessible through
113  * CVodeGetCurrentStep and CVodeGetErrWeights, respectively
114  * (see cvode.h). The unit roundoff is available as
115  * UNIT_ROUNDOFF defined in sundials_types.h.
116  *
117  * -----------------------------------------------------------------
118  */
119
