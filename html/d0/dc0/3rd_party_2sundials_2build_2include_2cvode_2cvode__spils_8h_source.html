73 /*
74  * -----------------------------------------------------------------
75  * Type : CVSpilsPrecSetupFn
76  * -----------------------------------------------------------------
77  * The user-supplied preconditioner setup function PrecSetup and
78  * the user-supplied preconditioner solve function PrecSolve
79  * together must define left and right preconditoner matrices
80  * P1 and P2 (either of which may be trivial), such that the
81  * product P1*P2 is an approximation to the Newton matrix
82  * M = I - gamma*J. Here J is the system Jacobian J = df/dy,
83  * and gamma is a scalar proportional to the integration step
84  * size h. The solution of systems P z = r, with P = P1 or P2,
85  * is to be carried out by the PrecSolve function, and PrecSetup
86  * is to do any necessary setup operations.
87  *
88  * The user-supplied preconditioner setup function PrecSetup
89  * is to evaluate and preprocess any Jacobian-related data
90  * needed by the preconditioner solve function PrecSolve.
91  * This might include forming a crude approximate Jacobian,
92  * and performing an LU factorization on the resulting
93  * approximation to M. This function will not be called in
94  * advance of every call to PrecSolve, but instead will be called
95  * only as often as necessary to achieve convergence within the
96  * Newton iteration. If the PrecSolve function needs no
97  * preparation, the PrecSetup function can be NULL.
98  *
99  * For greater efficiency, the PrecSetup function may save
100  * Jacobian-related data and reuse it, rather than generating it
101  * from scratch. In this case, it should use the input flag jok
102  * to decide whether to recompute the data, and set the output
103  * flag *jcurPtr accordingly.
104  *
105  * Each call to the PrecSetup function is preceded by a call to
106  * the RhsFn f with the same (t,y) arguments. Thus the PrecSetup
107  * function can use any auxiliary data that is computed and
108  * saved by the f function and made accessible to PrecSetup.
109  *
110  * A function PrecSetup must have the prototype given below.
111  * Its parameters are as follows:
112  *
113  * t is the current value of the independent variable.
114  *
115  * y is the current value of the dependent variable vector,
116  * namely the predicted value of y(t).
117  *
118  * fy is the vector f(t,y).
119  *
120  * jok is an input flag indicating whether Jacobian-related
121  * data needs to be recomputed, as follows:
122  * jok == FALSE means recompute Jacobian-related data
123  * from scratch.
124  * jok == TRUE means that Jacobian data, if saved from
125  * the previous PrecSetup call, can be reused
126  * (with the current value of gamma).
127  * A Precset call with jok == TRUE can only occur after
128  * a call with jok == FALSE.
129  *
130  * jcurPtr is a pointer to an output integer flag which is
131  * to be set by PrecSetup as follows:
132  * Set *jcurPtr = TRUE if Jacobian data was recomputed.
133  * Set *jcurPtr = FALSE if Jacobian data was not recomputed,
134  * but saved data was reused.
135  *
136  * gamma is the scalar appearing in the Newton matrix.
137  *
138  * user_data is a pointer to user data - the same as the user_data
139  * parameter passed to the CVodeSetUserData function.
140  *
141  * tmp1, tmp2, and tmp3 are pointers to memory allocated
142  * for N_Vectors which can be used by
143  * CVSpilsPrecSetupFn as temporary storage or
144  * work space.
145  *
146  * NOTE: If the user's preconditioner needs other quantities,
147  * they are accessible as follows: hcur (the current stepsize)
148  * and ewt (the error weight vector) are accessible through
149  * CVodeGetCurrentStep and CVodeGetErrWeights, respectively).
150  * The unit roundoff is available as UNIT_ROUNDOFF defined in
151  * sundials_types.h.
152  *
153  * Returned value:
154  * The value to be returned by the PrecSetup function is a flag
155  * indicating whether it was successful. This value should be
156  * 0 if successful,
157  * > 0 for a recoverable error (step will be retried),
158  * < 0 for an unrecoverable error (integration is halted).
159  * -----------------------------------------------------------------
160  */
161 
162 typedef int (*CVSpilsPrecSetupFn)(realtype t, N_Vector y, N_Vector fy,
163  booleantype jok, booleantype *jcurPtr,
164  realtype gamma, void *user_data,
165  N_Vector tmp1, N_Vector tmp2,
166  N_Vector tmp3);

168 /*
169  * -----------------------------------------------------------------
170  * Type : CVSpilsPrecSolveFn
171  * -----------------------------------------------------------------
172  * The user-supplied preconditioner solve function PrecSolve
173  * is to solve a linear system P z = r in which the matrix P is
174  * one of the preconditioner matrices P1 or P2, depending on the
175  * type of preconditioning chosen.
176  *
177  * A function PrecSolve must have the prototype given below.
178  * Its parameters are as follows:
179  *
180  * t is the current value of the independent variable.
181  *
182  * y is the current value of the dependent variable vector.
183  *
184  * fy is the vector f(t,y).
185  *
186  * r is the right-hand side vector of the linear system.
187  *
188  * z is the output vector computed by PrecSolve.
189  *
190  * gamma is the scalar appearing in the Newton matrix.
191  *
192  * delta is an input tolerance for use by PSolve if it uses
193  * an iterative method in its solution. In that case,
194  * the residual vector Res = r - P z of the system
195  * should be made less than delta in weighted L2 norm,
196  * i.e., sqrt [ Sum (Res[i]*ewt[i])^2 ] < delta.
197  * Note: the error weight vector ewt can be obtained
198  * through a call to the routine CVodeGetErrWeights.
199  *
200  * lr is an input flag indicating whether PrecSolve is to use
201  * the left preconditioner P1 or right preconditioner
202  * P2: lr = 1 means use P1, and lr = 2 means use P2.
203  *
204  * user_data is a pointer to user data - the same as the user_data
205  * parameter passed to the CVodeSetUserData function.
206  *
207  * tmp is a pointer to memory allocated for an N_Vector
208  * which can be used by PSolve for work space.
209  *
210  * Returned value:
211  * The value to be returned by the PrecSolve function is a flag
212  * indicating whether it was successful. This value should be
213  * 0 if successful,
214  * positive for a recoverable error (step will be retried),
215  * negative for an unrecoverable error (integration is halted).
216  * -----------------------------------------------------------------
217  */
218

220  N_Vector r, N_Vector z,

236  *

238  *
239  * Jv is the output N_Vector containing J*v.
240  *
241  * t is the current value of the independent variable.
242  *
243  * y is the current value of the dependent variable

245  *
246  * fy is the vector f(t,y).
247  *
248  * user_data is a pointer to user data, the same as the user_data
249  * parameter passed to the CVodeSetUserData function.

285  * the nonlinear iteration is multiplied to get a
