111 /*
112  * -----------------------------------------------------------------
113  * Enumerations for inputs to:
114  * CVodeCreate (lmm, iter),
115  * CVodeSensInit, CvodeSensinit1, CVodeSensReInit (ism),
116  * CVodeAdjInit (interp),
117  * CVode (itask)
118  * -----------------------------------------------------------------
119  * Symbolic constants for the lmm and iter parameters to CVodeCreate
120  * the input parameter itask to CVode, and the input parameter interp
121  * to CVodeAdjInit, are given below.
122  *
123  * lmm: The user of the CVODES package specifies whether to use
124  * the CV_ADAMS or CV_BDF (backward differentiation formula)
125  * linear multistep method. The BDF method is recommended
126  * for stiff problems, and the CV_ADAMS method is recommended
127  * for nonstiff problems.
128  *
129  * iter: At each internal time step, a nonlinear equation must
130  * be solved. The user can specify either CV_FUNCTIONAL
131  * iteration, which does not require linear algebra, or a
132  * CV_NEWTON iteration, which requires the solution of linear
133  * systems. In the CV_NEWTON case, the user also specifies a
134  * CVODE linear solver. CV_NEWTON is recommended in case of
135  * stiff problems.
136  *
137  * ism: This parameter specifies the sensitivity corrector type
138  * to be used. In the CV_SIMULTANEOUS case, the nonlinear
139  * systems for states and all sensitivities are solved
140  * simultaneously. In the CV_STAGGERED case, the nonlinear
141  * system for states is solved first and then, the
142  * nonlinear systems for all sensitivities are solved
143  * at the same time. Finally, in the CV_STAGGERED1 approach
144  * all nonlinear systems are solved in a sequence.
145  *
146  * itask: The itask input parameter to CVode indicates the job
147  * of the solver for the next user step. The CV_NORMAL
148  * itask is to have the solver take internal steps until
149  * it has reached or just passed the user specified tout
150  * parameter. The solver then interpolates in order to
151  * return an approximate value of y(tout). The CV_ONE_STEP
152  * option tells the solver to just take one internal step
153  * and return the solution at the point reached by that step.
154  *
155  * interp: Specifies the interpolation type used to evaluate the
156  * forward solution during the backward integration phase.
157  * CV_HERMITE specifies cubic Hermite interpolation.
158  * CV_POYNOMIAL specifies the polynomial interpolation
159  * -----------------------------------------------------------------
160  */

264 /*
265  * -----------------------------------------------------------------
266  * Type : CVRhsFn
267  * -----------------------------------------------------------------
268  * The f function which defines the right hand side of the ODE
269  * system y' = f(t,y) must have type CVRhsFn.
270  * f takes as input the independent variable value t, and the
271  * dependent variable vector y. It stores the result of f(t,y)
272  * in the vector ydot. The y and ydot arguments are of type
273  * N_Vector.
274  * (Allocation of memory for ydot is handled within CVODES)
275  * The user_data parameter is the same as the user_data
276  * parameter set by the user through the CVodeSetUserData routine.
277  * This user-supplied pointer is passed to the user's f function
278  * every time it is called.
279  *
280  * A CVRhsFn should return 0 if successful, a negative value if
281  * an unrecoverable error occured, and a positive value if a
282  * recoverable error (e.g. invalid y values) occured.
283  * If an unrecoverable occured, the integration is halted.
284  * If a recoverable error occured, then (in most cases) CVODES
285  * will try to correct and retry.
286  * -----------------------------------------------------------------
287  */
288

290  N_Vector ydot, void *user_data);

314  * -----------------------------------------------------------------
315  * Type : CVEwtFn
316  * -----------------------------------------------------------------
317  * A function e, which sets the error weight vector ewt, must have

330  * passed to the user's e function every time it is called.
331  * A CVEwtFn e must return 0 if the error weight vector has been
332  * successfuly set and a non-zero value otherwise.
333  * -----------------------------------------------------------------

591 /*
592  * -----------------------------------------------------------------
593  * Functions : CVodeSStolerances
594  * CVodeSVtolerances
595  * CVodeWFtolerances
596  * -----------------------------------------------------------------
597  *
598  * These functions specify the integration tolerances. One of them
599  * MUST be called before the first call to CVode.
600  *
601  * CVodeSStolerances specifies scalar relative and absolute tolerances.
602  * CVodeSVtolerances specifies scalar relative tolerance and a vector
603  * absolute tolerance (a potentially different absolute tolerance
604  * for each vector component).
605  * CVodeWFtolerances specifies a user-provides function (of type CVEwtFn)
606  * which will be called to set the error weight vector.
607  *
608  * The tolerances reltol and abstol define a vector of error weights,
609  * ewt, with components
610  * ewt[i] = 1/(reltol*abs(y[i]) + abstol) (in the SS case), or
611  * ewt[i] = 1/(reltol*abs(y[i]) + abstol[i]) (in the SV case).
612  * This vector is used in all error and convergence tests, which
613  * use a weighted RMS norm on all error-like vectors v:
614  * WRMSnorm(v) = sqrt( (1/N) sum(i=1..N) (v[i]*ewt[i])^2 ),
615  * where N is the problem dimension.
616  *
617  * The return value of these functions is equal to CV_SUCCESS = 0 if
618  * there were no errors; otherwise it is a negative int equal to:
619  * CV_MEM_NULL indicating cvode_mem was NULL (i.e.,
620  * CVodeCreate has not been called).
621  * CV_NO_MALLOC indicating that cvode_mem has not been

1165 /*
1166  * -----------------------------------------------------------------
1167  * Function : CVode
1168  * -----------------------------------------------------------------
1169  * CVode integrates the ODE over an interval in t.
1170  * If itask is CV_NORMAL, then the solver integrates from its
1171  * current internal t value to a point at or beyond tout, then
1172  * interpolates to t = tout and returns y(tout) in the user-
1173  * allocated vector yout. If itask is CV_ONE_STEP, then the solver
1174  * takes one internal time step and returns in yout the value of
1175  * y at the new internal time. In this case, tout is used only
1176  * during the first call to CVode to determine the direction of
1177  * integration and the rough scale of the problem. If tstop is
1178  * enabled (through a call to CVodeSetStopTime), then CVode returns
1179  * the solution at tstop. Once the integrator returns at a tstop
1180  * time, any future testing for tstop is disabled (and can be
1181  * reenabled only though a new call to CVodeSetStopTime).
1182  * The time reached by the solver is placed in (*tret). The
1183  * user is responsible for allocating the memory for this value.
1184  *
1185  * cvode_mem is the pointer to CVODES memory returned by
1186  * CVodeCreate.
1187  *
1188  * tout is the next time at which a computed solution is desired.
1189  *
1190  * yout is the computed solution vector. In CV_NORMAL mode with no
1191  * errors and no roots found, yout=y(tout).
1192  *
1193  * tret is a pointer to a real location. CVode sets (*tret) to
1194  * the time reached by the solver and returns yout=y(*tret).
1195  *
1196  * itask is CV_NORMAL or CV_ONE_STEP. These two modes are described above.
1197  *
1198  * Here is a brief description of each return value:
1199  *
1200  * CV_SUCCESS: CVode succeeded and no roots were found.
1201  *
1202  * CV_ROOT_RETURN: CVode succeeded, and found one or more roots.
1203  * If nrtfn > 1, call CVodeGetRootInfo to see
1204  * which g_i were found to have a root at (*tret).
1205  *
1206  * CV_TSTOP_RETURN: CVode succeded and returned at tstop.
1207  *
1208  * CV_MEM_NULL: The cvode_mem argument was NULL.
1209  *
1210  * CV_NO_MALLOC: cvode_mem was not allocated.
1211  *
