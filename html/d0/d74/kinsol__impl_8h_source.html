260 /*
261  * -----------------------------------------------------------------
262  * Function : int (*kin_linit)(KINMem kin_mem)
263  * -----------------------------------------------------------------
264  * kin_linit initializes solver-specific data structures (including
265  * variables used as counters or for storing statistical information),
266  * but system memory allocation should be done by the subroutine
267  * that actually initializes the environment for liner solver
268  * package. If the linear system is to be preconditioned, then the
269  * variable setupNonNull (type booleantype) should be set to TRUE
270  * (predefined constant) and the kin_lsetup routine should be
271  * appropriately defined.
272  *
273  * kinmem pointer to an internal memory block allocated during
274  * prior calls to KINCreate and KINMalloc
275  *
276  * If the necessary variables have been successfully initialized,
277  * then the kin_linit function should return 0 (zero). Otherwise,
278  * the subroutine should indicate a failure has occurred by
279  * returning a non-zero integer value.
280  * -----------------------------------------------------------------
281  */
282 
283 /*
284  * -----------------------------------------------------------------
285  * Function : int (*kin_lsetup)(KINMem kin_mem)
286  * -----------------------------------------------------------------
287  * kin_lsetup interfaces with the user-supplied pset subroutine (the
288  * preconditioner setup routine), and updates relevant variable
289  * values (see KINSpgmrSetup/KINSpbcgSetup). Simply stated, the
290  * kin_lsetup routine prepares the linear solver for a subsequent
291  * call to the user-supplied kin_lsolve function.
292  *
293  * kinmem pointer to an internal memory block allocated during
294  * prior calls to KINCreate and KINMalloc
295  *
296  * If successful, the kin_lsetup routine should return 0 (zero).
297  * Otherwise it should return a non-zero value.
298  * -----------------------------------------------------------------
299  */
300 
301 /*
302  * -----------------------------------------------------------------
303  * Function : int (*kin_lsolve)(KINMem kin_mem, N_Vector xx,
304  * N_Vector bb, realtype *sJpnorm, realtype *sFdotJp)
305  * -----------------------------------------------------------------
306  * kin_lsolve interfaces with the subroutine implementing the
307  * numerical method to be used to solve the linear system J*xx = bb,
308  * and must increment the relevant counter variable values in
309  * addition to computing certain values used by the global strategy
310  * and forcing term routines (see KINInexactNewton, KINLineSearch,
311  * KINForcingTerm, and KINSpgmrSolve/KINSpbcgSolve).
312  *
313  * kinmem pointer to an internal memory block allocated during
314  * prior calls to KINCreate and KINMalloc
315  *
316  * xx vector (type N_Vector) set to initial guess by kin_lsolve
317  * routine prior to calling the linear solver, but which upon
318  * return contains an approximate solution of the linear
319  * system J*xx = bb, where J denotes the system Jacobian
320  *
321  * bb vector (type N_Vector) set to -func(u) (negative of the
322  * value of the system function evaluated at the current
323  * iterate) by KINLinSolDrv before kin_lsolve is called
324  *
325  * sJpnorm holds the value of the L2-norm (Euclidean norm) of
326  * fscale*(J(u)*pp) upon return
327  *
328  * sFdotJp holds the value of the scaled F(u) (fscale*F) dotted
329  * with the scaled J(u)*pp vector upon return
330  *
331  * If successful, the kin_lsolve routine should return 0 (zero).
332  * Otherwise it should return a positive value if a re-evaluation
333  * of the lsetup function could recover, or a negative value if
334  * no such recovery is possible.
335  * -----------------------------------------------------------------
336  */
337
