55 /*---------------------------------------------------------------
56  Type: ARKDlsDenseJacFn
57 
58  A dense Jacobian approximation function Jac must be of type
59  ARKDlsDenseJacFn. Its parameters are:
60 
61  N is the problem size.
62 
63  Jac is the dense matrix (of type DlsMat) that will be loaded
64  by a ARKDlsDenseJacFn with an approximation to the Jacobian
65  matrix J = (df_i/dy_j) at the point (t,y).
66 
67  t is the current value of the independent variable.
68 
69  y is the current value of the dependent variable vector,
70  namely the predicted value of y(t).
71 
72  fy is the vector f(t,y).
73 
74  user_data is a pointer to user data - the same as the user_data
75  parameter passed to ARKodeSetFdata.
76 
77  tmp1, tmp2, and tmp3 are pointers to memory allocated for
78  vectors of length N which can be used by a ARKDlsDenseJacFn
79  as temporary storage or work space.
80 
81  A ARKDlsDenseJacFn should return 0 if successful, a positive
82  value if a recoverable error occurred, and a negative value if
83  an unrecoverable error occurred.
84 
85  NOTE: The following are two efficient ways to load a dense Jac:
86  (1) (with macros - no explicit data structure references)
87  for (j=0; j < Neq; j++) {
88  col_j = DENSE_COL(Jac,j);
89  for (i=0; i < Neq; i++) {
90  generate J_ij = the (i,j)th Jacobian element
91  col_j[i] = J_ij;
92  }
93  }
94  (2) (without macros - explicit data structure references)
95  for (j=0; j < Neq; j++) {
96  col_j = (Jac->data)[j];
97  for (i=0; i < Neq; i++) {
98  generate J_ij = the (i,j)th Jacobian element
99  col_j[i] = J_ij;
100  }
101  }
102  A third way, using the DENSE_ELEM(A,i,j) macro, is much less
103  efficient in general. It is only appropriate for use in small
104  problems in which efficiency of access is NOT a major concern.
105 
106  NOTE: If the user's Jacobian routine needs other quantities,
107  they are accessible as follows: hcur (the current stepsize)
108  and ewt (the error weight vector) are accessible through
109  ARKodeGetCurrentStep and ARKodeGetErrWeights, respectively
110  (see arkode.h). The unit roundoff is available as
111  UNIT_ROUNDOFF defined in sundials_types.h.
112 
113 ---------------------------------------------------------------*/
114 typedef int (*ARKDlsDenseJacFn)(long int N, realtype t,
115  N_Vector y, N_Vector fy,
116  DlsMat Jac, void *user_data,
117  N_Vector tmp1, N_Vector tmp2,
118  N_Vector tmp3);

153 /*---------------------------------------------------------------
154  Type: ARKDlsBandJacFn
155 
156  A band Jacobian approximation function Jac must have the
157  prototype given below. Its parameters are:
158 
159  N is the length of all vector arguments.
160 
161  mupper is the upper half-bandwidth of the approximate banded
162  Jacobian. This parameter is the same as the mupper parameter
163  passed by the user to the linear solver initialization function.
164 
165  mlower is the lower half-bandwidth of the approximate banded
166  Jacobian. This parameter is the same as the mlower parameter
167  passed by the user to the linear solver initialization function.
168 
169  t is the current value of the independent variable.
170 
171  y is the current value of the dependent variable vector,
172  namely the predicted value of y(t).
173 
174  fy is the vector f(t,y).
175 
176  Jac is the band matrix (of type DlsMat) that will be loaded
177  by a ARKDlsBandJacFn with an approximation to the Jacobian matrix
178  Jac = (df_i/dy_j) at the point (t,y).
179  Three efficient ways to load J are:
180 
181  (1) (with macros - no explicit data structure references)
182  for (j=0; j < n; j++) {
183  col_j = BAND_COL(Jac,j);
184  for (i=j-mupper; i <= j+mlower; i++) {
185  generate J_ij = the (i,j)th Jacobian element
186  BAND_COL_ELEM(col_j,i,j) = J_ij;
187  }
188  }
189 
190  (2) (with BAND_COL macro, but without BAND_COL_ELEM macro)
191  for (j=0; j < n; j++) {
192  col_j = BAND_COL(Jac,j);
193  for (k=-mupper; k <= mlower; k++) {
194  generate J_ij = the (i,j)th Jacobian element, i=j+k
195  col_j[k] = J_ij;
196  }
197  }
198 
199  (3) (without macros - explicit data structure references)
200  offset = Jac->smu;
201  for (j=0; j < n; j++) {
202  col_j = ((Jac->data)[j])+offset;
203  for (k=-mupper; k <= mlower; k++) {
204  generate J_ij = the (i,j)th Jacobian element, i=j+k
205  col_j[k] = J_ij;
206  }
207  }
208  Caution: Jac->smu is generally NOT the same as mupper.
209 
210  The BAND_ELEM(A,i,j) macro is appropriate for use in small
211  problems in which efficiency of access is NOT a major concern.
212 
213  user_data is a pointer to user data - the same as the user_data
214  parameter passed to ARKodeSetFdata.
215 
216  NOTE: If the user's Jacobian routine needs other quantities,
217  they are accessible as follows: hcur (the current stepsize)
218  and ewt (the error weight vector) are accessible through
219  ARKodeGetCurrentStep and ARKodeGetErrWeights, respectively
220  (see arkode.h). The unit roundoff is available as
221  UNIT_ROUNDOFF defined in sundials_types.h
222 
223  tmp1, tmp2, and tmp3 are pointers to memory allocated for
224  vectors of length N which can be used by a ARKDlsBandJacFn
225  as temporary storage or work space.
226 
227  A ARKDlsBandJacFn should return 0 if successful, a positive value
228  if a recoverable error occurred, and a negative value if an
229  unrecoverable error occurred.
230 ---------------------------------------------------------------*/
