63 /*
64  * -----------------------------------------------------------------
65  * Types : KINSlsSparseJacFn
66  * -----------------------------------------------------------------
67  *
68  * A sparse Jacobian approximation function jaceval must be of type
69  * KINSlsSparseJacFn.
70  * Its parameters are:
71  *
72  * u is the current value of the dependent variable vector
73  *
74  * fval is the residual vector F(tt,yy,yp).
75  *
76  * JacMat is the compressed sparse column matrix (of type SlsMat)
77  * to be loaded by an KINSlsSparseJacFn routine with an approximation
78  * to the system Jacobian matrix
79  * J = dF/dy
80  * Note that JacMat is NOT preset to zero!
81  * Matrix data is for the nonzero entries of the Jacobian are stored in
82  * compressed column format. Row indices of entries in
83  * column j are stored in J->data[colptrs[j]]
84  * through J->data[colptrs[j+i]-1]
85  * and corresponding numerical values of the Jacobian are stored
86  * in the same entries of a.
87  *
88  * user_data is a pointer to user Jacobian data
89  *
90  * vtemp1 and vtemp2 are pointers to memory allocated for
91  * N_Vectors which can be used by an KINSparseJacFn routine
92  * as temporary storage or work space.
93  *
94  * A KINSlsSparseJacFn should return
95  * 0 if successful,
96  * 1 if successful but the user code indicates a new factorization
97  * of the Jacobian is required
98  * a positive int if a recoverable error occurred, or
99  * a negative int if a nonrecoverable error occurred.
100  *
101  * -----------------------------------------------------------------
102  */
103
